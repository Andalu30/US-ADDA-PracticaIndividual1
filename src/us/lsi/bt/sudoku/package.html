El Problema del Sudoku problema clásico que puede ser resuelto por diversas técnicas algorítmicas.

El problema parte de una cuadrícula de tamaño N. Es decir con N filas y N columnas. La cuadrícula está subdividida en subcuadrículas de tamaño M que deber ser divisor de M. Hay, por lo tanto, N*N casillas y (N/M)*(N/M) subcuadrículas.

Cada casilla puede ser identificada por las coordenadas (x,y). Ambas toman valores en el rango [0,N). Una casilla puede ser identificada alternativamente por un entero p en el rango [0,N*N). El entero que se asocia a cada casilla se determina asignado 0 a la casilla (0,0), 1 a (1,0) y siguiendo una fila tras otra.

El objetivo del problema es colocar un número entero del intervalo [0,...,N) en cada casilla de tal forma que en cada fila, cada columna y cada subcuadrícula estén todos los valores posibles. Al principio se dan un conjunto de casillas con valores predeterminados

Los datos que aparecen en un Sodoku pueden ser modelados como un objeto del tipo {@link us.lsi.bt.sudoku.CuadroSudoku CuadroSudoku}

A un Problema Sudoku le podemos asociar el tipo {@link us.lsi.bt.sudoku.ProblemaSudoku ProblemaSudoku}

Las clases disponibles en el paquete son:

  • La clase {@link us.lsi.bt.sudoku.Casilla Casilla} representa las casillas.
  • La clase {@link us.lsi.sa.sudoku.CasillaAG CasillaAG} representa las casillas en un código similar
  • La clase {@link us.lsi.bt.sudoku.CuadroSudoku CuadroSudoku} representa una cuadrícula tamaño N*N con valores en las casillas.
  • La clase {@link us.lsi.sa.sudoku.CuadroSudokuAG CuadroSudokuAG} representa una cuadrícula tamaño N*N con valores en las casillas con un código simialr al anterior
  • La clase {@link us.lsi.bt.sudoku.ProblemaSodoku ProblemaSudoku} implementa los detalles de un Problema Sudoku
  • La clase {@link us.lsi.bt.sudoku.EstadoSudokuBT EstadoSudokuBT} implementa estado del problema para ser resuelto mediante Bactracking
  • La clase {@link us.lsi.bt.sudoku.EstadoSudokuSAAp1 EstadoSudokuSAAp1} implementa estado del problema para ser resuelto mediante Simulated Annealing
  • La clase {@link us.lsi.bt.sudoku.EstadoSudokuSAAp2 EstadoSudokuSAAp2} implementa estado del problema para ser resuelto mediante Simulated Annealing
  • La clase {@link us.lsi.bt.sudoku.ProblemaSudokuBT ProblemaSudokuBT} implementa un problema para ser resuelto mediante Bactracking
  • La clase {@link us.lsi.ag.sudoku.TestSudokuAG TestSudokuAG} el tes del problema para ser resuelto mediante Algoritmos Genéticos
  • La clase {@link us.lsi.sa.sudoku.ProblemaSudokuAG ProblemaSudokuAG} implementa un problema para ser resuelto mediante Simulated Annealing o Algoritmos Genéticos
